# バグ修正要件ドキュメント

## はじめに

プレビューパネルでスクロール中に、カーソルがダイアグラムコンテナ（Mermaid、PlantUML、SVG）上を通過すると、スクロールホイールイベントがダイアグラムのズームハンドラに奪われ、ページスクロールが停止してダイアグラムがズームされてしまう。これは閲覧体験を大きく損なうバグである。

ユーザーがダイアグラムを明示的にクリック/フォーカスした場合にのみズーム・パン操作が有効になるべきであり（GitHub方式のインタラクション）、通常のスクロール時はページスクロールがダイアグラムコンテナを素通りすべきである。

## バグ分析

### 現在の動作（不具合）

1.1 WHEN ユーザーがプレビューパネルをスクロール中にカーソルがダイアグラムコンテナ（`.diagram-container`）上を通過する THEN スクロールホイールイベントが `event.preventDefault()` により奪われ、ページスクロールが停止する
1.2 WHEN ユーザーがダイアグラムコンテナ上でスクロールホイールを操作する THEN 明示的なクリックやフォーカスなしにダイアグラムが意図せずズームされる
1.3 WHEN ユーザーがダイアグラムコンテナ上でマウスボタンを押してドラッグする THEN 明示的なフォーカスなしにダイアグラムがパンされる

### 期待される動作（正しい動作）

2.1 WHEN ユーザーがプレビューパネルをスクロール中にカーソルがダイアグラムコンテナ上を通過する THEN ページスクロールがダイアグラムコンテナを素通りし、通常通りスクロールが継続されるものとする
2.2 WHEN ユーザーがダイアグラムコンテナをクリックしてフォーカスした後にスクロールホイールを操作する THEN ダイアグラムがズームされるものとする
2.3 WHEN ユーザーがダイアグラムコンテナをクリックしてフォーカスした後にマウスドラッグする THEN ダイアグラムがパンされるものとする
2.4 WHEN フォーカスされたダイアグラムコンテナの外側をクリックする THEN ダイアグラムのフォーカスが解除され、スクロールホイールが通常のページスクロールに戻るものとする

### 変更されない動作（リグレッション防止）

3.1 WHEN ダイアグラムコンテナがフォーカスされた状態でスクロールホイールを操作する THEN ズーム機能は従来通り正しく動作し続けるものとする
3.2 WHEN ダイアグラムコンテナがフォーカスされた状態でマウスドラッグする THEN パン機能は従来通り正しく動作し続けるものとする
3.3 WHEN ダイアグラムコンテナがフォーカスされた状態でダブルクリックする THEN ズームリセット機能は従来通り正しく動作し続けるものとする
3.4 WHEN ダイアグラムコンテナ以外の領域でスクロールする THEN ページスクロールは従来通り正しく動作し続けるものとする
3.5 WHEN ズームインジケーターが表示される THEN ズーム倍率の表示は従来通り正しく動作し続けるものとする

## バグ条件の導出

### バグ条件関数

```pascal
FUNCTION isBugCondition(X)
  INPUT: X of type WheelEvent on .diagram-container
  OUTPUT: boolean

  // ダイアグラムコンテナがフォーカスされていない状態でホイールイベントが発生した場合にバグが発生する
  RETURN X.target is inside .diagram-container AND container is NOT focused
END FUNCTION
```

### プロパティ仕様

```pascal
// プロパティ: 修正チェック - 非フォーカス時のスクロール透過
FOR ALL X WHERE isBugCondition(X) DO
  result ← handleWheel'(X)
  ASSERT event.preventDefault() is NOT called
  AND page scroll continues normally
END FOR
```

### 保存目標

```pascal
// プロパティ: 保存チェック - フォーカス時のズーム動作維持
FOR ALL X WHERE NOT isBugCondition(X) DO
  ASSERT F(X) = F'(X)
  // フォーカスされたダイアグラムコンテナでのズーム・パン動作は変更前と同一
END FOR
```
