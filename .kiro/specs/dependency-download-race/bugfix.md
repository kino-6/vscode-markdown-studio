# バグ修正要件ドキュメント

## はじめに

拡張機能のアクティベーション時に `ensureAll()` が依存関係（Corretto JDK および Playwright Chromium）を非同期でダウンロードするが、コマンド（PDF エクスポート、PlantUML レンダリングなど）はダウンロード完了を待たずに実行可能な状態になっている。これにより、依存関係が未準備の状態でコマンドが実行され、サイレントな失敗や不明瞭なエラーが発生する。

根本原因は、`activate()` 関数内で `ensureAll()` を `await` しているものの、コマンド登録は `ensureAll()` の完了後に行われるため一見問題ないように見えるが、実際には `ensureAll()` が失敗した場合やバイナリが存在しない場合でもコマンドは登録・実行される。さらに、コマンド実行時に依存関係の準備状態を確認するゲート機構が存在しないため、依存関係が必要なコマンドが未準備状態で実行されてしまう。

## バグ分析

### 現在の動作（不具合）

1.1 WHEN 依存関係のダウンロードが失敗した状態で「Export PDF」コマンドが実行された場合 THEN システムは Chromium が見つからないという不明瞭なエラーメッセージを表示し、ユーザーに根本原因が伝わらない

1.2 WHEN 依存関係のダウンロードが失敗した状態で PlantUML を含む Markdown をプレビューした場合 THEN システムは Java が利用できないという不明瞭なエラーを PlantUML ブロック内に表示し、依存関係セットアップとの関連が不明確である

1.3 WHEN `ensureAll()` が部分的に成功した状態（例：Corretto は成功、Chromium は失敗）でコマンドが実行された場合 THEN システムはどの依存関係が不足しているかを明示せず、コマンドごとに異なる不明瞭なエラーを返す

1.4 WHEN 依存関係のセットアップ中にユーザーが「Setup Dependencies」コマンドを再実行した場合 THEN システムは並行してインストールを開始し、競合状態が発生する可能性がある

### 期待される動作（正しい動作）

2.1 WHEN 依存関係のダウンロードが未完了または失敗した状態で「Export PDF」コマンドが実行された場合 THEN システムは「依存関係をインストール中です。しばらくお待ちください...」のような明確なメッセージを表示し、依存関係の準備完了後にコマンドを実行するか、セットアップの再実行を案内する SHALL

2.2 WHEN 依存関係のダウンロードが未完了または失敗した状態で PlantUML を含む Markdown をプレビューした場合 THEN システムは Java 依存関係が未準備であることを明確に示し、「Setup Dependencies」コマンドの実行を案内する SHALL

2.3 WHEN `ensureAll()` が部分的に成功した状態でコマンドが実行された場合 THEN システムはそのコマンドに必要な特定の依存関係が不足していることを明示し、適切な対処方法を案内する SHALL

2.4 WHEN 依存関係のセットアップ中にユーザーが「Setup Dependencies」コマンドを再実行した場合 THEN システムは既にセットアップが進行中であることを通知し、重複実行を防止する SHALL

2.5 WHEN 依存関係を必要としないコマンド（プレビュー表示など、PlantUML を含まない場合）が実行された場合 THEN システムは依存関係の状態に関係なく即座にコマンドを実行する SHALL

### 変更されない動作（リグレッション防止）

3.1 WHEN すべての依存関係が正常にインストール済みの状態で「Export PDF」コマンドが実行された場合 THEN システムは従来通り正常に PDF をエクスポートする SHALL CONTINUE TO

3.2 WHEN すべての依存関係が正常にインストール済みの状態で PlantUML を含む Markdown をプレビューした場合 THEN システムは従来通り正常に PlantUML ダイアグラムをレンダリングする SHALL CONTINUE TO

3.3 WHEN PlantUML を含まない通常の Markdown をプレビューした場合 THEN システムは従来通り即座にプレビューを表示する SHALL CONTINUE TO

3.4 WHEN 「Validate Environment」コマンドが実行された場合 THEN システムは従来通り環境の検証結果を表示する SHALL CONTINUE TO

3.5 WHEN 拡張機能がアクティベートされた場合 THEN システムは従来通り `ensureAll()` を呼び出して依存関係の自動セットアップを試みる SHALL CONTINUE TO

3.6 WHEN 依存関係のインストールが完了した後にコマンドが実行された場合 THEN システムは追加の遅延なく従来通りコマンドを実行する SHALL CONTINUE TO
